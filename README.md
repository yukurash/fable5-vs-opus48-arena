# Fable 5 vs Opus 4.8 — 実アプリ開発で比較する

Claude Fable 5 と Claude Opus 4.8 に**まったく同じプロンプトを1回だけ**渡してWebアプリ等を作らせ、成果物そのもので比較した検証のデータ・成果物置き場です。

**🎮 全成果物を実際に触れます → [デモ一覧](https://yukurash.github.io/fable5-vs-opus48-arena/)**

## 検証ルール

- 同一プロンプト・**1ターン・人間の介入なし**(Claude Code ヘッドレスモード `claude -p`)
- 各ランは凍結した同一の開始状態(gitブランチ `base`)から実施
- 成果物はモデルに触らせず、ハーネスが機械的にコミット
- モデルは互いの存在も、採点基準(隠しテスト等)も知らない
- コストはAPI料金換算値(`claude -p` の出力する `total_cost_usd`)

## 種目と結果

### Task A: 仕様書からの REST API 実装(客観採点)

[仕様書](tasks/taskA-spec.md)(ToDo管理API、エッジケース多数)を渡して実装させ、モデルが知らない[隠し受け入れテスト27件](tasks/taskA-hidden-tests/)で採点。

| | Fable 5 | Opus 4.8 |
|---|---|---|
| 隠しテスト | **27/27** | **27/27** |
| 実時間 | 337s | 386s |
| コスト | $2.23 | $1.62 |
| 備考 | `.gitignore` を自発的に作成、可搬性の高い依存選択 | モジュール分割は丁寧、`__pycache__` をコミット混入 |

### Task C-1: 押してはいけないボタン(自由演技)

[3行のプロンプト](prompts/c1.md)だけ渡して丸投げ。発想力と演出の勝負。各モデル2ラン。

| | Fable 5 run1 | Fable 5 run2 | Opus 4.8 run1 | Opus 4.8 run2 |
|---|---|---|---|---|
| 実時間 | 427s | 408s | 197s | 115s |
| コスト | $2.77 | $2.49 | $0.60 | $0.36 |
| 規模 | 3ファイル/1,297行 | 3ファイル/1,411行 | 1ファイル/672行 | 1ファイル/412行 |

### Task C-2: Win95風デスクトップ(規定演技)

[要件14項目](tasks/taskC2-spec.md)(ウィンドウシステム、ペイント、マインスイーパー、メモ帳、ゴミ箱…)を実装させて機能チェックリストで採点。詳細: [checklist-c2.md](results/checklist-c2.md)

| | Fable 5 run1 | Fable 5 run2 | Opus 4.8 run1 |
|---|---|---|---|
| 機能スコア | **0/14** ⚠️ | **14/14** | **14/14** |
| 実時間 | 981s | 1,664s | 772s |
| コスト | $5.64 | $10.75 | $3.16 |

⚠️ Fable 5 run1 は起動処理を担う `shell.js` を**書き忘れたまま納品**し、BIOS風起動画面から先に進めない([スクリーンショット](results/screenshots/c2-fable5-boot.png))。書かれた2,327行の品質自体は高いが、成果物としては動かない。run2 では再現せず、ウェルカムダイアログ付きの完動品を納品した。

### 合計(全9ラン)

| | Fable 5(5ラン) | Opus 4.8(4ラン) |
|---|---|---|
| 完走率(動く成果物の納品) | 4/5 | 4/4 |

## リポジトリ構成

```
prompts/      投入プロンプト原文(これがモデルへの指示のすべて)
tasks/        仕様書と隠し受け入れテスト
results/      メトリクスCSV・採点結果・スクリーンショット
artifacts/    全ランの成果物(無修正。動かないものも証拠としてそのまま)
index.html    デモランチャー(GitHub Pages)
```

## 留意事項

- ラン数は各タスク1〜2回(n が小さいため、傾向の示唆であって統計的結論ではない)
- 実行環境: Claude Code 2.1.172 / Windows 11 (ARM64) / 2026-06-11 実施
- Task C-2 の採点は「コード静的解析 + puppeteer による起動実証」による(詳細は checklist-c2.md)
- 成果物は各ランのgitブランチからの無修正コピー(`__pycache__` のみ除外)。ただしデモでブラウザのタブから判別できるよう、各 `index.html` の `<title>` にだけモデル名(`[Fable 5]` / `[Opus 4.8]`)を前置しています。完全に無改変の原本は各作業リポジトリ(`arena-task-*`)のブランチにあります
